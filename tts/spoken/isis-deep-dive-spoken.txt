IS-IS Deep Dive. Module 2, Section 1 of the Service Provider Study Guide.

IS-IS is the IGP of choice for the vast majority of service provider backbones worldwide. Unlike OSPF, it runs directly over Layer 2 with no IP dependency, scales elegantly with multi-level hierarchy, and carries new address families via TLV extensibility without protocol rewrites. If you work in service provider networking, you work in IS-IS. Period.

After this section you will understand how SPF-based IGPs build topology and select best paths, which timers and fast-fail mechanisms control convergence behavior, how to validate IGP health with adjacency, LSDB, and route checks, and common failure modes and how to isolate them quickly.

Why IS-IS Over OSPF in Service Provider Networks?

Number one: Protocol-independent by design. IS-IS runs on CLNS at Layer 2, not IP. Adding IPv6, Segment Routing, or any new address family is just a new TLV — not a new protocol version.

Number two: Simpler area design. Two levels versus OSPF's stub, NSSA, and totally-stubby madness. Level 1 and Level 2 is clean.

Number three: TLV extensibility. This is the killer feature. IS-IS has been extended for Traffic Engineering, Segment Routing, IPv6, and more — all without breaking backward compatibility. OSPF needed OSPFv3 just for IPv6.

Number four: Stability at scale. SPF calculation is bounded per level. Flooding is naturally contained.

Number five: Industry inertia. Every major service provider runs IS-IS. Operational tooling, tribal knowledge, and vendor optimization all favor it.

IS-IS Fundamentals: Addressing.

IS-IS uses a NET, or Network Entity Title, for addressing. The format is: AFI, followed by Area ID, followed by System ID, followed by the selector.

AFI 49 means private addressing, which is what most SP deployments use. The Area ID is variable length, 1 to 13 bytes — keep it simple, like 49 dot something. The System ID is 6 bytes and must be unique. A common convention is to encode the loopback IP address. The selector is always zero-zero for a router.

Levels and Hierarchy.

Level 1 is intra-area, analogous to an OSPF non-backbone area. Level 2 is inter-area and forms the backbone, analogous to OSPF Area 0. A Level 1 slash Level 2 router participates in both, similar to an OSPF ABR.

Here's the key difference from OSPF: there's no "Area 0" requirement. Level 2 forms the backbone by connecting Level 1 slash Level 2 routers. Any L1/L2 router is automatically a backbone member.

In SP reality, most cores run Level 2 only everywhere. The L1/L2 hierarchy is used for large access and aggregation layers. Don't over-engineer levels where flat L2 works.

PDU Types.

IS-IS uses four PDU types. IIH, or Hello packets, handle neighbor discovery and adjacency maintenance. LSPs carry link-state information — that's the database. CSNP, the Complete Sequence Number PDU, handles database synchronization. And PSNP, the Partial Sequence Number PDU, handles requests and acknowledgments.

Adjacency Formation.

The process goes like this: A router sends an IIH on its interface. The neighbor receives it and checks for area match at Level 1, authentication, and MTU. Then a three-way handshake occurs as defined in RFC 5303: the states go from Down, to Init, to Up. Once the adjacency is Up, routers exchange LSPs via CSNP and PSNP.

For link types: Point-to-point has no DIS election and uses direct LSP exchange. Use this for ALL service provider links — even Ethernet links. Set the circuit type to point-to-point. Broadcast mode elects a DIS based on highest priority then highest SNPA. The DIS is NOT a designated router like OSPF — there's no backup, and preemption happens. It generates a pseudonode LSP.

TLV Architecture.

This is where IS-IS really shines. Everything is a Type-Length-Value. TLV type 1 carries Area Addresses. Type 22 carries Extended IS Reachability for TE-capable neighbor info. Type 135 carries Extended IP Reachability for IPv4 prefixes with TE metrics. Type 236 carries IPv6 Reachability. Type 242 carries Router Capability for Segment Routing and Flex-Algo.

Why does this matter? When Segment Routing was added to IS-IS, it was just new sub-TLVs under existing TLVs. No protocol version change. No new packet types. OSPF needed extensions across both version 2 and version 3.

SPF Optimization.

BFD, or Bidirectional Forwarding Detection, provides sub-second failure detection, typically 50 to 300 milliseconds. It runs independently of IS-IS timers. Always deploy BFD on service provider core links.

For timer tuning: Hello interval defaults to 10 seconds, aggressive settings bring it down to 1 to 3 seconds. The Hello multiplier defaults to 3 — don't lower it or you'll get false positives. SPF initial wait is 50 milliseconds, secondary wait is 200 milliseconds with exponential backoff, and max wait caps at 5 seconds. RFC 8405 defines this SPF back-off algorithm to prevent SPF storms during convergence events.

Prefix prioritization ensures critical prefixes like loopbacks and BGP next-hops get SPF'd first. This ensures BGP next-hops converge before less-critical routes. On IOS-XR use "spf prefix-priority" under router isis. On Junos use "spf-options priority" under protocols isis.

The Overload Bit.

Setting the overload bit in the LSP header tells other routers: don't transit through me, but my directly-connected prefixes are still reachable. Use cases include maintenance windows for graceful traffic draining, boot-up delays to wait for BGP to converge before attracting transit traffic, and memory or CPU protection. This is a critical operational tool.

Multi-Topology.

Multi-topology, or MT, runs separate SPF computations for IPv4 and IPv6. This allows ships-in-the-night routing with different topologies per address family. Use it during dual-stack migrations where IPv4 and IPv6 topologies differ.

Key Configuration Recommendations.

Always use wide metrics — this is required for Traffic Engineering and Segment Routing. Non-negotiable. Set circuit type to Level 2 only in the core to reduce unnecessary Level 1 adjacencies. Configure point-to-point on all inter-router links — no DIS, faster convergence. Disable hello padding after adjacency is up to save bandwidth. And set overload on startup with wait-for-BGP to prevent blackholing during boot.

Verification and Monitoring.

For health checks, verify all adjacencies are Up with expected hold time. Ensure LSDB size is consistent across all routers — same number of LSPs per level. SPF run count should be stable, not continuously incrementing. No overload bit should be set unexpectedly. And sequence numbers should increment normally without wrapping.

Troubleshooting Common Issues.

If an adjacency is stuck in Init state, the most likely causes are MTU mismatch or one-way connectivity. Check the interface for MTU settings and verify Layer 2 connectivity.

If an adjacency is flapping, look for an unstable link, timer mismatch, or BFD issues. Check interface error counters and verify hello and hold timers match.

If routes are missing, check for area mismatch or metric overflow. Verify the NET area configuration and ensure wide metrics are enabled.

If you see SPF storms — constant SPF recalculations — look for a flapping link or LSP corruption. Check IS-IS statistics to find the churning LSP.

If you see LSP purge storms, you likely have a system ID conflict. Every router must have a unique system ID.

The troubleshooting methodology is straightforward. Step 1: check adjacencies. Step 2: check the LSDB for missing LSPs or sequence number anomalies. Step 3: check SPF statistics for run count and recent triggers. Step 4: check the interface for correct level, metric, and network type. Step 5: check the wire — BFD state, interface counters, Layer 2 connectivity. Step 6: packet capture with Wireshark, filtering on IS-IS.

War Story: The Phantom DIS.

A large carrier had intermittent traffic blackholes on a metro Ethernet ring. Root cause: one link was configured as broadcast, the default, instead of point-to-point. The DIS election was creating a pseudonode, and when the DIS flapped, all routers on the segment had to reconverge through the pseudonode LSP — adding 2 to 3 seconds of convergence time. The fix: point-to-point on every inter-router link. Always. Forever.
