# 9.1 — SP Transport Hierarchy

## Theory

### The Three-Tier Transport Model

Service provider networks are built in layers. The transport hierarchy defines three physical/optical tiers, each with distinct reach, capacity, and design constraints:

```
┌─────────────────────────────────────────────────────────────────────┐
│                        LONG-HAUL / BACKBONE                        │
│   Reach: 500–10,000+ km  |  Capacity: 10–100+ Tbps per fiber pair │
│   Amplified DWDM spans   |  Coherent 400G/800G per wavelength     │
│   ROADMs at major nodes   |  Regen sites every 500–3,000 km *     │
├─────────────────────────────────────────────────────────────────────┤
│                          METRO / REGIONAL                          │
│   Reach: 50–500 km       |  Capacity: 1–40 Tbps per fiber pair    │
│   Ring/mesh topologies    |  DWDM or CWDM                         │
│   ROADMs at hub sites     |  100G/400G per wavelength              │
│   Aggregation + grooming  |  OTN switching optional                │
├─────────────────────────────────────────────────────────────────────┤
│                          ACCESS / LAST MILE                        │
│   Reach: 0–80 km         |  Capacity: 1G–100G per customer        │
│   Point-to-point or PON   |  Direct detect or low-cost coherent   │
│   Fiber, microwave, xDSL  |  Minimal amplification                │
│   CPE/demarcation devices |  Often grey optics (single wavelength) │
└─────────────────────────────────────────────────────────────────────┘
```

**Key principle**: Each tier has different cost-per-bit economics, protection requirements, and technology choices. What works at 50 km doesn't work at 5,000 km. What's cost-effective for 10G access is absurd for 400G backbone.

### Access Tier

The access tier connects customer premises to the nearest SP aggregation point (typically called a **Central Office (CO)** or **Point of Presence (PoP)**).

#### Access Technologies

| Technology | Typical Reach | Capacity | Use Case |
|-----------|--------------|----------|----------|
| Dark fiber | 0–80 km | 1G–400G+ | Enterprise, DC interconnect |
| Active Ethernet (P2P) | 0–80 km | 1G–100G | Business services |
| GPON/XGS-PON | 0–20 km (split dependent *) | 2.5G/10G shared | Residential FTTH |
| 25G/50G PON | 0–20 km | 25G/50G shared | Next-gen residential/business |
| Microwave | 0–50 km (LOS) | 1–10G | Cell tower backhaul, rural |
| xDSL (VDSL2, G.fast) | 0–1 km | 100M–1G | Legacy copper last mile |
| Fixed wireless (mmWave) | 0–2 km | 1–10G | Dense urban, 5G FWA |

#### Access Design Patterns

**Point-to-Point Fiber** — One fiber pair per customer. Simple, predictable, no sharing. Preferred for enterprise/wholesale. The customer gets a dedicated wavelength or Ethernet handoff.

**Passive Optical Network (PON)** — One fiber from the CO splits passively (1:32 or 1:64) to serve multiple customers. Cost-effective for residential. The OLT (Optical Line Terminal) at the CO manages time-division or wavelength-division upstream access. *Note: PON reach is heavily split-ratio dependent. A 1:64 split adds ~18 dB of loss, making 20 km marginal for GPON. At high split ratios, practical reach may be 10–15 km.*

**Microwave/Fixed Wireless** — Where fiber isn't economical. Common for cell tower backhaul in rural areas. Subject to rain fade, atmospheric absorption, and line-of-sight requirements.

#### Access Demarcation

The **demarcation point** is where the SP's responsibility ends and the customer's begins:

```
Customer Site          Demarcation          SP Access Network
┌──────────┐     ┌─────────────────┐     ┌──────────────────┐
│  CE/CPE  │────▶│  NID / NTE / ONT│────▶│  OLT / Agg Switch │
│          │     │  (smart SFP,    │     │  at CO/PoP        │
│          │     │   media converter│     │                    │
│          │     │   or ONT)       │     │                    │
└──────────┘     └─────────────────┘     └──────────────────┘
```

- **NID (Network Interface Device)**: Demarcation box, often with loopback testing
- **NTE (Network Termination Equipment)**: Smart demarcation with OAM (802.1ag, Y.1731)
- **ONT (Optical Network Terminal)**: PON customer-side device

### Metro Tier

The metro tier aggregates access traffic and transports it between PoPs within a metropolitan area or region. This is where optical networking gets interesting.

#### Metro Topologies

**Ring** — The workhorse of metro transport. Simple protection (switch to opposite direction in <50 ms), well-understood capacity planning. Most metro DWDM deployments start as rings.

```
        PoP-A ──────── PoP-B
       /                    \
      /                      \
   PoP-D                   PoP-C
      \                      /
       \                    /
        PoP-E ──────── PoP-F
         (Hub / CO)
```

**Hub-and-spoke** — Simpler, cheaper, but single point of failure at the hub. Common for smaller metros or access aggregation within a metro.

**Mesh** — Partial or full mesh between major PoPs. Used in dense metros where traffic patterns are many-to-many. Requires ROADMs for wavelength routing flexibility.

#### Metro DWDM

Metro DWDM systems typically operate on the **C-band** (1530–1565 nm) with 48, 80, or 96 channels at 50 GHz or 100 GHz fixed spacing. Modern systems increasingly use **flex-grid** (ITU-T G.694.1 flexible frequency grid with 6.25 GHz granularity), which allows variable-width channels — essential for 400G+ super-channels that may occupy 75 GHz or wider slots. Newer **C+L band** systems nearly double capacity by adding the L-band (1565–1625 nm) — the L-band provides roughly 80–90% of C-band capacity due to slightly worse amplifier noise figure.

Key metro DWDM components:

| Component | Function | Typical Location |
|-----------|----------|-----------------|
| Transponder/Muxponder | Client signal → DWDM wavelength | Each PoP (terminal) |
| OADM (fixed) | Add/drop fixed wavelengths | Small ring sites |
| ROADM | Add/drop any wavelength dynamically | Major PoPs |
| EDFA (amplifier) | Boost signal strength | Every 80–120 km |
| OSC (Optical Supervisory Channel) | Management/monitoring | Out-of-band, per span |

Metro spans are typically **unamplified** (< 80 km between nodes) or require only one or two amplifiers. This keeps costs down compared to long-haul.

#### Metro Protection

| Scheme | Recovery Time | Capacity Cost | Description |
|--------|--------------|---------------|-------------|
| OCh-SNCP (UPSR) | < 50 ms | 2× (dedicated protect) | Unidirectional path-switched ring (optical channel level) |
| OMS-SPRing (BLSR) | < 50 ms | ~1.5× (shared protect) | Bidirectional, shared protection (optical multiplex section) |
| ROADM restoration | 100–500 ms | 1× + spare capacity | Software-controlled wavelength reroute |
| IP/MPLS FRR | < 50 ms | N/A (layer 3) | TI-LFA at packet layer |

**Design insight**: Modern networks increasingly rely on IP-layer protection (TI-LFA, Module 6.3) rather than optical protection. This trades slightly slower failover for dramatically better capacity efficiency — you don't need to reserve 50% of optical capacity for protection paths that rarely activate.

### Long-Haul Tier

The backbone. Long-haul transport connects cities, countries, and continents. Distances from 500 km to 6,000+ km terrestrial (submarine systems reach 10,000+ km but use fundamentally different designs — different amplifier spacing (~50–70 km), shore-powered repeaters, specialized SLTE equipment, and repair timelines measured in weeks. Submarine is its own discipline and out of scope for this section).

#### Long-Haul Architecture

```
City A                                                          City B
┌─────┐                                                        ┌─────┐
│ROADM│──Span──[EDFA]──Span──[EDFA]──Span──[Regen]──Span──[EDFA]──│ROADM│
│     │  ~80km         ~80km         ~80km    site   ~80km         │     │
└─────┘                                                        └─────┘
        ◀────────── Amplified Section ─────────▶ ◀── New Section──▶
        ◀──────────────── ~500-3,000+ km ────────────────────────────▶
                  (transparent reach — modulation dependent *)
```

**Amplifier spacing**: EDFAs (Erbium-Doped Fiber Amplifiers) every 60–120 km, depending on fiber type, span loss, and channel count. Each EDFA introduces ASE (Amplified Spontaneous Emission) noise that accumulates over distance. **Distributed Raman Amplification (DRA)** is commonly deployed alongside EDFAs in long-haul systems — it uses the transmission fiber itself as the gain medium by pumping backward from each amplifier site, improving OSNR by 3–6 dB compared to EDFA-only designs.

**Regeneration**: After 500–3,000+ km (depending on modulation format, fiber quality, and channel loading), the optical signal-to-noise ratio (OSNR) degrades below usable levels. A **regenerator** (or "3R regen" — re-amplify, re-shape, re-time) converts the signal back to electrical, cleans it up, and retransmits optically. Modern coherent DSP has pushed regen-free reach significantly.

**Transparent reach** depends on:
- Modulation format (QPSK goes farther than 16QAM/64QAM, but carries fewer bits)
- Fiber type (G.652 vs G.654.E low-loss fiber)
- Channel spacing and count (more channels = more nonlinear interference)
- Amplifier noise figure
- Span loss

#### The Capacity-Reach Trade-off

This is the fundamental law of optical transport:

| Modulation | Bits/Symbol (dual-pol) | Capacity per λ | Typical Reach † |
|-----------|-------------|---------------|---------------|
| DP-QPSK | 4 | 100–200G | 3,000–6,000 km |
| DP-8QAM | 6 | 200–300G | 1,500–3,000 km |
| DP-16QAM | 8 | 400G | 500–1,500 km |
| DP-32QAM | 10 | 600G | 120–400 km |
| DP-64QAM | 12 | 800G | 80–150 km |

*† Reach values assume moderate channel loading on modern fiber with typical amplifier spacing. Best-case (low channel count, G.654.E fiber, Raman assist) can exceed these ranges. Worst-case (fully loaded, older G.652 fiber, no Raman) may fall below.*

**DP = Dual Polarization** — uses both X and Y polarizations of light to double capacity.

Higher-order modulation packs more bits into each symbol but requires better OSNR (signal quality). This means shorter reach before regeneration. The optical planner's job is choosing the right modulation for each route's distance.

#### Long-Haul Design Constraints

**Fiber characterization matters**: Chromatic dispersion (CD), polarization mode dispersion (PMD), and nonlinear effects (self-phase modulation, cross-phase modulation, four-wave mixing) all degrade signal quality over distance. Modern coherent DSP compensates for CD and PMD digitally. FWM is largely mitigated on modern dispersion-unmanaged fiber with coherent detection, but SPM and XPM remain significant. The fundamental limit is the trade-off between ASE noise (too little launch power) and nonlinear interference (too much launch power) — finding the optimal launch power per channel is a core optical engineering task.

**Route diversity**: Long-haul routes must be physically diverse. Two "diverse" paths that share a common conduit for even 1 km aren't diverse at all. Physical diversity is planned at the duct/conduit level, not the logical level.

**Right-of-way (ROW)**: Long-haul fiber follows existing infrastructure — railroad tracks, highways, pipelines, power transmission corridors. This is why fiber routes sometimes take geographically bizarre paths.

### The Protocol Stack

Understanding how layers map to the transport hierarchy:

```
┌─────────────────────────────────┐
│  Layer 3: IP/MPLS/SRv6         │  ← Modules 2-8
│  (Routing, forwarding, VPNs)    │
├─────────────────────────────────┤
│  Layer 2: Ethernet              │  ← Module 8 (EVPN)
│  (Switching, bridging)          │
├─────────────────────────────────┤
│  Layer 1.5: OTN (optional)      │  ← Section 9.3
│  (Digital wrapper, multiplexing)│
├─────────────────────────────────┤
│  Layer 1: DWDM                  │  ← Section 9.2
│  (Wavelength multiplexing)      │
├─────────────────────────────────┤
│  Layer 0: Fiber optic           │  ← Physical glass
│  (Photons, amplification)       │
└─────────────────────────────────┘
```

**The trend**: Collapsing layers. The industry is moving toward **IP-over-DWDM** (eliminating OTN) using pluggable coherent optics (ZR/ZR+) directly in routers. More on this in Section 9.4.

### Capacity Planning

#### Fiber Pair Economics

A single fiber pair (one fiber each direction) in a long-haul DWDM system with 96 C-band channels at 400G per channel:

- **Raw capacity**: Up to 96 × 400G = 38.4 Tbps per direction (note: 400G channels typically require 75 GHz flex-grid slots, reducing usable count to ~64 channels in standard C-band at 50 GHz fixed grid; 96 × 400G assumes flex-grid or extended C-band)
- **With C+L band**: ~80 Tbps per direction
- **Per-wavelength cost** drops as you fill the system — the expensive part is the fiber, amplifiers, and ROADMs (shared infrastructure), not the individual transponders

#### Planning Rules of Thumb

| Metric | Access | Metro | Long-Haul |
|--------|--------|-------|-----------|
| Fiber pairs per route | 1–2 | 2–12 | 4–48+ |
| Wavelengths per pair | 1 (grey) or 8–16 (CWDM) | 48–96 (DWDM) | 80–96+ (DWDM) |
| Per-λ capacity (2025+) | 10G–100G | 100G–400G | 100G–800G |
| Amplifier spacing | N/A or rare | 80–120 km | 60–100 km |
| Protection | Dual-homed | Ring/mesh | Diverse routes |
| Latency (propagation) | Negligible | ~5 μs/km × route km | ~5 μs/km × route km (route selection may be latency-driven) |
| Dominant cost | Fiber installation | ROADM nodes | Fiber + amplifiers |
| Typical vendor | Calix, ADTRAN | Ciena, Nokia, Infinera | Ciena, Nokia, Infinera, Huawei |

```
│  │  PE  │    │  P   │    │  P   │    │  P   │    │  PE  │     │

│     │    Transport / Optical Domain     │           │          │
│  ┌──┴───┐    ┌──┴───┐    ┌──┴───┐    ┌──┴───┐    ┌──┴───┐    │
│  │ROADM │───▶│ EDFA │───▶│ROADM │───▶│ EDFA │───▶│ROADM │    │

│  │Txpdr │    │      │    │      │    │      │    │Txpdr │    │
│  └──────┘    └──────┘    └──────┘    └──────┘    └──────┘    │

└───────────────────────────────────────────────────────────────┘
```

**Two operational domains**: Most SPs have separate IP/MPLS and transport/optical teams with different NMS/OSS systems. This is changing with converged IP+optical planning tools, but the organizational divide is real and affects troubleshooting.

### Multilayer Design Decisions

| Decision | Option A | Option B | Trade-off |
|----------|----------|----------|-----------|
| Protection layer | Optical (ROADM restoration) | IP (TI-LFA/FRR) | Speed vs efficiency |
| Grooming | OTN switching | IP/MPLS | Granularity vs simplicity |
| Transponder type | Line-side (chassis) | Pluggable (in router) | Flexibility vs integration |
| Capacity upgrade | Add wavelengths | Light new fiber pair | Incremental vs capex |
| Monitoring | Optical PM (pre-FEC BER) | IP counters | Proactive vs reactive |

### The IP-over-DWDM Debate

**Traditional model**: Router → grey optic → transponder → DWDM line system → transponder → grey optic → Router. Multiple conversion steps, multiple management planes, but each layer optimized independently.

**Converged model**: Router → pluggable coherent optic (ZR/ZR+) → DWDM line system → pluggable coherent optic → Router. Eliminates standalone transponders, reduces latency, simplifies management. But the router now "owns" the optical performance, blurring team boundaries.

**Reality (2026)**: Most SPs use a hybrid. Pluggable 400ZR for metro/DCI (< 120 km) and OpenZR+ for extended metro/regional (up to ~400 km), with traditional transponders for long-haul where performance margins matter. The full convergence is happening but isn't complete.

## Design Considerations

### When to Invest in Each Tier

**Access**: Build when customer demand justifies it. FTTH for residential (20-year asset), dark fiber for enterprise (high-margin, quick payback). PON for density, P2P for premium.

**Metro**: Build when aggregation traffic exceeds 10G between sites. ROADM rings when you need wavelength flexibility. Fixed filters for point-to-point overlays. Often the first place SPs deploy DWDM.

**Long-haul**: Build or lease based on traffic growth forecasts. New fiber builds cost $20K–$60K per route-mile in the US (varies wildly by terrain, permitting, existing conduit). Leasing IRU (Indefeasible Right of Use) wavelengths or dark fiber from carriers like Zayo, Lumen, or Crown Castle is common.

### Common Mistakes

1. **Designing optical and IP independently** — The best transport design considers IP topology, traffic matrices, and failure scenarios together. A beautiful ring topology means nothing if the IP layer sends all traffic through one node.

2. **Ignoring fiber characterization** — Not all fiber is equal. Old G.652 fiber has different dispersion and loss characteristics than modern G.654.E. PMD on older fiber can be a showstopper for 400G+ per wavelength.

3. **Over-engineering protection** — Running 1+1 optical protection AND IP-layer FRR means you're paying for protection twice. Pick a primary protection layer and use the other as backup.

4. **Forgetting about the amplifier chain** — Adding a wavelength to an existing DWDM system isn't free if the amplifiers are already running near saturation. Each new channel slightly degrades all existing channels (power rebalancing).

5. **Assuming "diverse" means diverse** — Two fibers in the same conduit, two conduits on the same bridge, two cables in the same ocean trench. Physical diversity requires physical verification, not logical assumptions.

### Emerging Trends (2025–2026)

| Trend | Impact | Timeline |
|-------|--------|----------|
| 800G per wavelength | Double metro/DCI capacity | Deploying now |
| Pluggable ZR+ (400G) | Eliminate metro transponders | Mainstream 2025+ |

## Verification & Monitoring

### Optical Performance Metrics

| Metric | What It Measures | Healthy Range | Alarm Threshold |
|--------|-----------------|---------------|-----------------|
| Optical power (dBm) | Signal strength | Per design (typ. -10 to +3) | ±3 dB from baseline |
| OSNR (dB) | Signal-to-noise ratio | > 15–20 dB (modulation dependent) | < design minimum |
| Pre-FEC BER | Bit errors before correction | < 1e-2 to 2.4e-2 (FEC dependent) | Approaching FEC threshold |
| Post-FEC BER | Bit errors after correction | 0 (error-free) | Any errors |
| CD (ps/nm) | Chromatic dispersion | Within DSP compensation range | Exceeds DSP range |
| PMD (ps) | Polarization mode dispersion | < 10–30 ps (rate dependent) | Exceeds tolerance |
| Channel power deviation | Power flatness across channels | < ±1 dB | > ±2 dB |

## Troubleshooting

### Common Issues

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| Sudden link down | Fiber cut (construction, rodents) | Locate break with OTDR, reroute via protection |
| Gradual BER increase | Connector degradation, fiber aging | Clean/replace connectors, OTDR test |
| Intermittent errors | Loose connector, vibration | Reseat/replace affected patch cord |
| All channels degrade simultaneously | Amplifier failure or fiber bend | Check EDFA alarms, inspect fiber route |
| Single channel degrades | Transponder issue or filter drift | Swap transponder, check ROADM filter |
| Post-maintenance errors | Fiber swapped (TX/RX reversed) | Verify patch panel connections |
| Power fluctuations | ROADM rebalancing after add/drop | Wait for stabilization, check VOA settings |

### The IP Engineer's Optical Troubleshooting Checklist

When the optical team says "it's fine" but your IP links keep flapping:

1. **Check interface error counters** — CRC errors, FCS errors, input errors on the router
2. **Check light levels** — `show interface <x> phy` or `show controllers optics <x>` (IOS-XR)
3. **Ask for pre-FEC BER** — If it's > 1e-4, something's wrong even if post-FEC is clean
4. **Check for micro-cuts** — Brief (< 50 ms) optical interruptions that the IP layer detects as flaps but optical monitoring may miss
5. **Verify fiber assignment** — After any maintenance, confirm the right fiber pair is connected
6. **Temperature** — Optics modules have operating temperature ranges. Check `show environment` and optics temperature

### War Story: The Backhoe That Tested Everything

*A Tier 1 SP had a "fully diverse" long-haul route between two major cities — Path A through the northern corridor (following a railway), Path B through the southern corridor (following a highway). Both paths were lit with 80-channel DWDM, running at 80% capacity.*

*One morning, a construction crew hit a gas line, which caused a fire, which burned through a conduit containing... both "diverse" fiber paths. They converged to cross the same river bridge — the only crossing point within 40 miles.*

*The fix took 72 hours. The root cause wasn't the backhoe — it was that nobody had verified physical diversity at the conduit/structure level. The fiber vendor's "diverse path" documentation showed different routes on different maps, at different scales, but nobody had walked the actual path.*

*Lesson: Diversity isn't a line on a map. It's a physical attribute of physical infrastructure. If you haven't verified it at the duct level (or better, the structure level), you don't have it. Post-incident, this SP implemented mandatory GIS-level diversity verification for all "protected" routes, with a minimum separation distance of 20 miles at all points.*

## Quick Reference

### Key Standards
- **ITU-T G.652**: Standard single-mode fiber
- **ITU-T G.654.E**: Low-loss, large-effective-area fiber (terrestrial long-haul; submarine uses G.654.B/C/D)
- **ITU-T G.694.1**: DWDM frequency/wavelength grid
- **ITU-T G.709**: OTN framing (Section 9.3)
- **ITU-T G.798**: OTN equipment functional blocks
- **ITU-T G.872**: OTN architecture
- **OIF-400ZR**: 400G pluggable coherent (interoperable)
- **OpenZR+**: Extended 400G pluggable (vendor-optimized)
- **OpenConfig**: Vendor-neutral YANG models for transport equipment

### Key Terminology
| Term | Meaning |
|------|---------|
| λ (lambda) | Wavelength — a single color of light carrying one channel |
| DWDM | Dense Wavelength Division Multiplexing — many λs on one fiber |
| CWDM | Coarse WDM — fewer channels, wider spacing, cheaper |
| EDFA | Erbium-Doped Fiber Amplifier — boosts all channels simultaneously |
| ROADM | Reconfigurable Optical Add-Drop Multiplexer — software-defined λ routing |
| OSNR | Optical Signal-to-Noise Ratio — the "health metric" of a wavelength |
| FEC | Forward Error Correction — mathematical error recovery |
| Transponder | Converts client signal (grey) to DWDM line signal (colored) |
| Muxponder | Multiplexes multiple client signals onto one DWDM wavelength |
| Coherent | Detection method using local oscillator (enables 100G+) |
| Direct detect | Simpler detection (NRZ ~25G/lane, PAM4 ~50–100G/lane) |
| OTN | Optical Transport Network — digital wrapper with OAM |
| OTDR | Optical Time-Domain Reflectometer — fiber fault locator |
| dBm | Decibels relative to 1 milliwatt — optical power unit |
| Span | Fiber segment between two amplification/termination points |
| Regen | Regeneration site — electrical cleanup of optical signal |
| IRU | Indefeasible Right of Use — long-term fiber/wavelength lease |

### Cert Relevance
- **IE-SP**: Transport hierarchy understanding expected; optical details less tested but appear in design scenarios
- **IE-SP**: Similar — focus on packet-optical integration and transport LSP concepts (Junos PTX)
- Neither cert expects you to be an optical engineer, but both expect you to understand the layer beneath your routers
