# 9.5 — Coherent Optics (400G+)

## Overview
> Coherent optical technology is the reason your 100G/400G/800G wavelengths exist. It replaced the old on-off keying (OOK) world where you either had a photon or you didn't, with a system that encodes data in the amplitude, phase, and polarization of light — simultaneously. This is the technology that turned fiber from a dumb pipe into a programmable medium. This section goes deep on coherent detection, digital signal processing (DSP), modulation formats from DP-QPSK through probabilistically-shaped DP-64QAM, the 400ZR/ZR+ ecosystem that's killing standalone transponders, 800G and beyond (1.6T), and the operational reality of tuning coherent parameters in production networks. If you touch anything beyond 10G in an SP network, you're touching coherent optics — whether you know it or not.

## Theory

### Why Coherent?

Before coherent, optical transmission used **direct detection** — intensity modulation with on-off keying (IM-OOK). A "1" was light on, a "0" was light off. Simple, cheap, and utterly limited:

| Parameter | Direct Detection (IM-OOK) | Coherent Detection |
|-----------|--------------------------|-------------------|
| Information encoded in | Amplitude only | Amplitude + Phase + Polarization |
| Bits per symbol | 1 | 2–12+ (depending on modulation) |
| Spectral efficiency | ~1 bit/s/Hz | 2–10+ bit/s/Hz |
| Dispersion tolerance | ~80 km @ 10G (requires DCF) | Thousands of km (DSP compensated) |
| PMD tolerance | Limited (requires fiber selection) | Compensated digitally |
| Max practical rate | 10G per wavelength | 800G+ per wavelength |
| OSNR sensitivity | Poor | Excellent (close to Shannon limit) |

Direct detection hit the wall at 10G. The CD tolerance at 10G IM-OOK is ~1600 ps/nm — roughly 96 km of G.652 fiber. At 40G, it drops to ~100 ps/nm (6 km without DCF). The physics won't let you go faster without coherent.

### Coherent Detection Fundamentals

Coherent detection recovers the full electric field of the received signal — both amplitude and phase — by beating it against a **local oscillator (LO)** laser:

```
                    ┌────────────────────┐
                    │  Coherent Receiver  │
                    │                     │
 Signal ──────────►│  ┌──────────────┐  │
 (from fiber)      │  │   90° Optical │  │──► XI  ──┐
                    │  │    Hybrid     │  │──► XQ  ──┤
 LO Laser ────────►│  │              │  │──► YI  ──┤──► ADCs ──► DSP
 (local)           │  │              │  │──► YQ  ──┘
                    │  └──────────────┘  │
                    │                     │
                    │  4 balanced         │
                    │  photodetectors     │
                    └────────────────────┘
```

**The 90° Optical Hybrid** splits signal and LO into four paths:
- **XI**: X-polarization, In-phase component
- **XQ**: X-polarization, Quadrature component (90° shifted)
- **YI**: Y-polarization, In-phase component
- **YQ**: Y-polarization, Quadrature component

Each component drives a **balanced photodetector pair** → 4 analog signals → 4 high-speed **ADCs** (analog-to-digital converters) → **DSP**.

This gives you access to the full electric field: `E(t) = A(t) · e^(jφ(t))` on both polarizations. Direct detection only sees `|E(t)|²` — you lose phase information entirely.

### Digital Signal Processing (DSP)

The DSP ASIC is the brain of every coherent transceiver. It performs miracles that were optically impossible:

```
ADC Input ──► Static     ──► Adaptive    ──► Carrier    ──► Symbol
              Equalization   Equalization    Recovery       Decision
              (CD comp.)     (PMD comp.)     (freq+phase)   (demapping)
              
              ┌──────────────────────────────────────────────────┐
              │            DSP Processing Pipeline               │
              │                                                  │
              │  1. Deskew & IQ imbalance correction             │
              │  2. Chromatic dispersion compensation (FIR)      │
              │  3. Clock recovery (Gardner/Mueller-Müller)      │
              │  4. Adaptive equalization (CMA → DD-LMA)         │
              │     └─ Compensates: PMD, PDL, filter narrowing   │
              │  5. Frequency offset estimation (FFT-based)      │
              │  6. Carrier phase recovery (Viterbi, BPS)        │
              │  7. Symbol decision + soft-decision FEC decode   │
              │  8. Performance monitoring (pre-FEC BER, OSNR)   │
              │                                                  │
              └──────────────────────────────────────────────────┘
```

**Key DSP capabilities:**

| Function | What It Does | Why It Matters |
|----------|-------------|----------------|
| CD compensation | Removes chromatic dispersion digitally | Eliminates DCF modules entirely |
| PMD compensation | Tracks and removes differential group delay | No more PMD-limited fiber links |
| PDL mitigation | Compensates polarization-dependent loss | Tolerates ROADM cascade effects |
| Nonlinear compensation | Partially reverses fiber nonlinearities | Extends reach at higher launch power |
| FEC | Soft-decision forward error correction | Pre-FEC BER of 2×10⁻² → post-FEC < 10⁻¹⁵ |
| Performance monitoring | Pre-FEC BER, OSNR estimation, CD/PMD | Real-time link quality without external OSA |

**DSP power consumption** is the dominant factor in coherent module design. A 400G ZR QSFP-DD module's DSP burns ~12-15W of the total ~18-20W. This is why DSP ASIC process node (7nm, 5nm) directly determines module form factor and density.

### Modulation Formats

Coherent uses **quadrature amplitude modulation (QAM)** on two polarizations — dual-polarization (DP) or polarization-multiplexed (PM):

```
     DP-QPSK              DP-16QAM             DP-64QAM
   (2 bits/pol)          (4 bits/pol)          (6 bits/pol)
   = 4 bits/sym          = 8 bits/sym          = 12 bits/sym
   
    Q                     Q                     Q
    │  ●   ●              │●  ●  ●  ●           │● ● ● ● ● ● ● ●
    │                     │●  ●  ●  ●           │● ● ● ● ● ● ● ●
    ┼──────── I           │●  ●  ●  ●           │● ● ● ● ● ● ● ●
    │                     │●  ●  ●  ●           │● ● ● ● ● ● ● ●
    │  ●   ●              ┼──────────── I       │● ● ● ● ● ● ● ●
                                                │● ● ● ● ● ● ● ●
   4 symbols             16 symbols             │● ● ● ● ● ● ● ●
   OSNR: ~12 dB          OSNR: ~18 dB           │● ● ● ● ● ● ● ●
                                                ┼────────────────── I
                                               64 symbols
                                               OSNR: ~24 dB
```

**Modulation comparison table (per wavelength, single carrier, ~60 GBd class):**

| Modulation | Bits/Symbol | Client Rate @ ~60 GBd | Spectral Eff. | Typical Reach | OSNR Req. (0.1nm) |
|-----------|------------|----------------------|--------------|--------------|-------------------|
| DP-QPSK | 4 | ~200G | ~3 bit/s/Hz | 3,000+ km | ~10-12 dB |
| DP-8QAM | 6 | ~300G | ~4.5 bit/s/Hz | 1,500 km | ~16 dB |
| DP-16QAM | 8 | ~400G | ~6 bit/s/Hz | 600-1,000 km | ~18-19 dB |
| DP-32QAM | 10 | ~500G | ~7.5 bit/s/Hz | 200-400 km | ~22 dB |
| DP-64QAM | 12 | ~540G | ~9 bit/s/Hz | 80-120 km | ~25-28 dB |

> **Reading this table**: All rows use the same ~60 GBd baud rate (typical for current-gen DSPs like 400ZR-class). Raw bit rate = baud × bits/symbol (e.g., 60 GBd × 8 = 480 Gbit/s for DP-16QAM). Client rate is lower due to FEC overhead (~15-25%). Higher modulation packs more bits per symbol but requires better OSNR and shorter reach. To deliver 400G at DP-QPSK, you'd need ~125 GBd — that's why long-haul 400G uses higher baud rate DSPs.
>
> **Note**: DP-8QAM is rarely deployed as a discrete format in practice — PCS fills the gap between QPSK and 16QAM by shaping a 16QAM constellation to achieve any rate in between.

> **The fundamental trade-off**: Higher modulation order = more capacity per wavelength = shorter reach. You can't cheat Shannon.

### Probabilistic Constellation Shaping (PCS)

Modern coherent DSPs don't use uniform QAM constellations. **Probabilistic Constellation Shaping (PCS)** transmits inner constellation points (lower amplitude) more frequently than outer points:

```
Uniform 64QAM              Shaped 64QAM (PCS)
                            
● ● ● ● ● ● ● ●           · · · · · · · ·
● ● ● ● ● ● ● ●           · · ○ ○ ○ ○ · ·
● ● ● ● ● ● ● ●           · ○ ○ ● ● ○ ○ ·
● ● ● ● ● ● ● ●           · ○ ● ● ● ● ○ ·
● ● ● ● ● ● ● ●           · ○ ● ● ● ● ○ ·
● ● ● ● ● ● ● ●           · ○ ○ ● ● ○ ○ ·
● ● ● ● ● ● ● ●           · · ○ ○ ○ ○ · ·
● ● ● ● ● ● ● ●           · · · · · · · ·

All points equally likely    Inner points transmitted more often
                            ● = frequent  ○ = medium  · = rare
```

**Benefits:**
- **1-1.5 dB** OSNR gain (equivalent to ~20-25% reach extension) at same bit rate
- **Continuous rate adaptation** — not locked to discrete QAM orders. Can hit any rate between, say, 200G and 800G on a single wavelength by adjusting shaping distribution
- **Approaches Shannon limit** more closely (~0.5 dB gap vs ~1.5 dB for uniform QAM)

PCS is what enables **flexible rate** operation. Instead of jumping from 16QAM (400G) to QPSK (200G) when a path gets longer, PCS smoothly reduces rate: 400G → 350G → 300G → 250G → 200G. Every dB of margin is converted to usable capacity.

**Vendor implementations:**
| Vendor | DSP ASIC | PCS Support | Notes |
|--------|----------|-------------|-------|
| Ciena | WaveLogic 5e/6 | Yes (FlexRate) | Pioneered PCS in commercial systems |
| Infinera | ICE6/ICE-X | Yes (Instant Bandwidth) | Multi-carrier + PCS |
| Nokia | PSE-6s | Yes (Probabilistic Rate) | Also supports super-channel bonding |
| Acacia (Cisco) | Coherent DSP (CIM 8) | Yes | In QSFP-DD/CFP2 form factors |
| Marvell (Inphi) | Canopus/Spica | Yes (400ZR+) | Industry-standard pluggable DSP |

### Baud Rate and Multi-Carrier

Two approaches to achieving 800G+ per wavelength:

**1. Single-carrier high baud rate:**
```
 ◄────── ~130 GHz ──────►
 ┌────────────────────────┐
 │    Single 800G          │
 │    ~130 GBd DP-16QAM    │
 └────────────────────────┘
         λ₁

 Pro: One wavelength, simpler
 Con: Extreme DSP complexity, wider spectrum
```

**2. Multi-carrier (sub-carrier/super-channel):**
```
 ◄──── ~75 GHz ────►◄──── ~75 GHz ────►
 ┌──────────────────┐┌──────────────────┐
 │  Sub-carrier 1    ││  Sub-carrier 2    │
 │  400G ~69 GBd     ││  400G ~69 GBd     │
 └──────────────────┘└──────────────────┘
         λ₁.a                λ₁.b

 Pro: Lower baud rate per carrier, better OSNR
 Con: Two laser sources, spectral guard bands
```

At 800G, both approaches are viable:
- **Cisco/Acacia**: Single-carrier ~130 GBd (requires 150 GHz channel spacing)
- **Infinera ICE-X**: Dual-carrier ~69 GBd each (fits 150 GHz)
- **Ciena WaveLogic 6**: Single-carrier ~100 GBd with PCS

At **1.6T**, multi-carrier is likely necessary — current DSP technology can't economically reach ~250+ GBd single-carrier in a practical form factor.

### FEC in Coherent Systems

Coherent optics use **soft-decision FEC (SD-FEC)** — the decoder receives probability information from the DSP, not just hard 0/1 decisions:

| FEC Type | Net Coding Gain | Pre-FEC BER Threshold | Overhead | Used In |
|----------|----------------|----------------------|----------|---------|
| HD-FEC (RS) | ~6.2 dB | 3.8×10⁻³ | ~7% | Legacy 10G/40G |
| SD-FEC (concatenated) | ~9.4 dB | 1.5×10⁻² | ~15% | Gen 1-2 coherent |
| cFEC (OIF 400ZR standard) | ~9.4 dB | ~4.5×10⁻³ | ~15% | 400ZR |
| oFEC (OpenZR+ MSA standard) | ~11.1 dB | ~1.25×10⁻² | ~15% | OpenZR+ |
| SD-FEC (proprietary) | ~11.5-12.5 dB | 2.4×10⁻² | ~20-27% | Vendor line systems |

**cFEC** (concatenated FEC) deserves special attention — it's the standard FEC for 400ZR, defined by OIF:
- Concatenated code: Hamming(128,119) inner + Staircase outer
- ~14.8% net coding overhead on the line side
- Enables multi-vendor interop (same FEC = same pre-FEC BER threshold)
- **KP4-FEC** (RS(544,514), IEEE 802.3 clause 119) is used on the **host-side** electrical interface between module and router ASIC — it is NOT concatenated with cFEC on the line side. This is a common point of confusion.

> **Key insight**: FEC overhead trades bandwidth for reach. 400ZR's cFEC with ~15% coding overhead means a 400G payload uses ~460G line rate. Proprietary line-system SD-FEC runs 20-27% overhead for maximum reach — more correction power, but sacrifices interop.

### The 400ZR/ZR+ Ecosystem

The most transformative development in SP optical networking since coherent itself:

**400ZR (OIF Implementation Agreement):**
- **Form factor**: QSFP-DD, OSFP (pluggable into router)
- **Data rate**: 400G (fixed)
- **Modulation**: DP-16QAM (fixed)
- **FEC**: cFEC (concatenated, interoperable per OIF IA)
- **Reach**: ~120 km (unamplified, point-to-point)
- **Grid**: 75 GHz fixed channel spacing
- **Wavelength**: C-band, 75 GHz ITU-T grid (~64 channels)
- **Power**: ~15-20W
- **Target**: DCI, metro P2P links
- **Multi-vendor**: ✅ Yes — that's the whole point

**ZR+ (OpenZR+ MSA):**
- **Form factor**: QSFP-DD, OSFP, CFP2-DCO
- **Data rate**: 100G, 200G, 300G, 400G (flexible)
- **Modulation**: DP-QPSK to DP-16QAM (selectable)
- **FEC**: oFEC (base) + vendor-enhanced SD-FEC options
- **Reach**: 500-2,000+ km (amplified, over line system)
- **Grid**: Flex-grid (variable channel spacing)
- **Power**: ~18-25W (more DSP headroom)
- **Target**: Metro, regional, some long-haul
- **Multi-vendor**: Partial — oFEC base mode interops, enhanced modes may not

```
                    400ZR               OpenZR+
                    ┌──────────┐        ┌──────────┐
 Application:       DCI/Metro P2P      Metro/Regional/Long-haul
 Rate:              400G fixed         100-400G flexible
 Modulation:        DP-16QAM fixed     QPSK-16QAM selectable
 Reach:             ~120 km            500-2000+ km
 Amplification:     None needed        Over amplified line system
 Interop:           Guaranteed         oFEC base: yes. Enhanced: varies
 Form factor:       QSFP-DD/OSFP      QSFP-DD/OSFP/CFP2-DCO
                    └──────────┘        └──────────┘
```

**What killed the standalone transponder** (for many use cases):
1. Router pluggable does the same job at lower cost
2. No shelf space, no separate management, no gray optic in between
3. Same QSFP-DD port can run gray 400G or coherent 400ZR — just swap the optic
4. IPoDWDM with router-integrated coherent eliminates an entire layer

**But transponders survive when:**
- You need >400G per wavelength (800G line-side with 400GE client)
- Long-haul reach exceeds ZR+ capability
- Muxponder ratios needed (10×100G client → 1×800G line)
- Regen/OEO required at intermediate sites
- Alien wavelength over 3rd-party line system needs tuning flexibility

### 800G and Beyond

The state of the art (2025-2026):

| Generation | Line Rate | Baud Rate | Modulation | Form Factor | Status |
|-----------|----------|-----------|------------|-------------|--------|
| 400ZR | 400G | ~60 GBd | DP-16QAM | QSFP-DD/OSFP | Production |
| 800G (line system) | 800G | ~90-130 GBd | DP-16/32QAM + PCS | CFP2-DCO | Production |
| 800ZR | 800G | ~125-130 GBd | DP-16QAM | OSFP-XD | Early 2026 |
| 800G pluggable | 800G | ~100+ GBd | DP-16QAM + PCS | OSFP-XD | Sampling |
| 1.6T (line system) | 1.6T | 2×~100 GBd | Multi-carrier PCS-64QAM | - | Lab/demo |

**Key challenges at 800G+:**
1. **Power**: 800G pluggable DSP needs ~25-35W — pushes OSFP thermal limits
2. **Bandwidth**: ~125-130 GBd requires ultra-wideband analog components (TIAs, drivers, modulators)
3. **Spectral width**: Single-carrier 800G at ~130 GBd needs ~150 GHz spacing — reduces wavelength count per fiber
4. **Nonlinear penalty**: Higher baud rate + higher launch power = more fiber nonlinearity
5. **ADC resolution**: Higher QAM orders need higher ENOB (effective number of bits) — 6-8 bit ADCs at 130+ GSa/s

> **Platform note**: Cisco 8000 uses 5-tuple interface naming (R/S/I/P/L) — e.g., `controller Optics0/0/0/0/0`. NCS 5500/5700 uses 4-tuple. Always verify your platform's naming convention before applying configs.

**OSFP-XD** (eXtra Dense): The emerging form factor for 800G pluggables. 25% larger faceplate than OSFP, supports ~35W module power. Required because QSFP-DD can't cool an 800G coherent DSP.

### Coherent vs. Direct Detect for 100G and Below

Not everything needs coherent. The decision framework:

| Criteria | Use Direct Detect | Use Coherent |
|----------|------------------|-------------|
| Distance | < 10 km | > 10 km (or > 2 km at 400G) |
| Rate per λ | ≤ 100G | > 100G |
| Dispersion | Negligible | Any significant CD or PMD |
| Cost sensitivity | Extreme (DC interconnect) | Moderate (metro/WAN) |
| Wavelength grid | LAN-WDM (CWDM spacing) | DWDM (50/75/100 GHz) |
| Example | 100G-DR1/DR4, 400G-DR4 | 400ZR, ZR+, line system |

**PAM4** (4-level pulse amplitude modulation) bridges the gap:
- 100G per lane using ~53 GBd PAM4 (2 bits/symbol)
- 400G-DR4 = 4 × 100G PAM4 lanes on 4 fibers
- No phase/polarization recovery — much simpler (and cheaper) than coherent
- But distance-limited: ~2 km at 400G without amplification

## Architecture

### Coherent Module Architecture

Inside a QSFP-DD 400ZR module:

```
  ┌──────────────────────────────────────────────────────────┐
  │                    QSFP-DD Module                         │
  │                                                          │
  │  ┌──────────┐   ┌──────────┐   ┌──────────────────────┐ │
  │  │ Tunable   │   │ Modulator │   │                      │ │
  │  │ Laser     │──►│ (MZM)     │──►│                      │ │
  │  │ (C-band)  │   │ DP-IQ     │   │     Fiber            │ │
  │  │           │   └──────────┘   │     Interface         │ │
  │  │           │                   │     (LC duplex)       │ │
  │  │           │   ┌──────────┐   │                      │ │
  │  │           │──►│ Coherent  │◄──│                      │ │
  │  │    (LO)   │   │ Receiver  │   │                      │ │
  │  └──────────┘   │ (ICR)     │   └──────────────────────┘ │
  │                  └────┬─────┘                             │
  │                       │ 4× analog                        │
  │                  ┌────▼─────┐                             │
  │                  │ DSP ASIC  │◄──── Host interface         │
  │                  │ (FEC,     │      (8×50G PAM4 or        │
  │                  │  equal.,  │       4×100G PAM4)         │
  │                  │  mapping) │                             │
  │                  └──────────┘                             │
  │                                                          │
  │  Power: ~15-20W        Temp: 0-70°C (commercial)         │
  └──────────────────────────────────────────────────────────┘
```

**Key components:**
- **Tunable laser**: Serves as both transmit source and local oscillator (split internally). C-band tunable across ~64 channels (75 GHz grid for 400ZR).
- **DP-IQ Modulator**: Dual-nested Mach-Zehnder modulator. Creates the DP-16QAM signal. Uses InP or thin-film LiNbO₃.
- **Integrated Coherent Receiver (ICR)**: 90° hybrid + 4 balanced photodetectors. Recovers XI, XQ, YI, YQ.
- **DSP ASIC**: The expensive part. 7nm or 5nm process. Performs all equalization, FEC, mapping.
- **Host interface**: Electrical interface to the router ASIC — typically 8×50G or 4×100G PAM4 SerDes lanes.

### Operational Modes and Line Rates

Modern coherent pluggables support multiple **operational modes** — pre-defined combinations of modulation, FEC, baud rate, and line rate:

```
 ┌───────────────────────────────────────────────────────┐
 │              Operational Mode Selection                │
 │                                                       │
 │  Mode     Modulation    Baud    Client    Reach       │
 │  ────     ──────────    ────    ──────    ─────       │
 │  0x01     DP-16QAM     ~60G    400GE     120 km      │  ← 400ZR
 │  0x20     DP-QPSK      ~30G    100GE     2000 km     │  ← ZR+ 100G
 │  0x21     DP-QPSK      ~60G    200GE     1200 km     │  ← ZR+ 200G
 │  0x22     DP-8QAM      ~42G    300GE     800 km      │  ← ZR+ 300G
 │  0x23     DP-16QAM     ~60G    400GE     500 km      │  ← ZR+ 400G
 │                                                       │
 │  Selection: OpenConfig terminal-device                │
 │  /terminal-device/logical-channels/channel/           │
 │     optical-channel/config/operational-mode           │
 └───────────────────────────────────────────────────────┘
```

> **Operational mode IDs** are defined per vendor in the OpenConfig catalog (openconfig-terminal-device). The OIF assigns standard mode IDs for 400ZR. ZR+ modes are MSA-defined but vary by vendor for enhanced FEC modes.

### What Good Looks Like

```
✅ Good coherent link:
  Pre-FEC BER:  < 1×10⁻³  (well within FEC threshold)
  Post-FEC BER: 0 (or < 10⁻¹⁵)
  OSNR:         > 22 dB (for DP-16QAM)
  Q-Factor:     > 8 dB
  CD:           Within DSP range (±50,000 ps/nm typical)
  DGD:          < 30 ps (most links < 5 ps)
  Rx Power:     Within optic spec (-22 to -10 dBm typical)

⚠️ Warning signs:
  Pre-FEC BER:  1×10⁻³ to 1×10⁻² (still correctable, but degrading)
  OSNR:         18-22 dB (margin shrinking)
  Q-Factor:     6-8 dB

❌ Trouble:
  Pre-FEC BER:  > 1.5×10⁻² (approaching FEC cliff)
  Post-FEC BER: > 0 (errors leaking through FEC — immediate action)
  OSNR:         < 18 dB (DP-16QAM will fail)
  Sudden CD change: Fiber route changed? Patch panel swap?
```

## Troubleshooting

### Common Issues

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| Link won't come up | Wrong frequency (both ends mismatch) | Verify wavelength/frequency matches on both sides |
| Link won't come up | Operational mode mismatch | Both ends must use same mode (400ZR mode 1 on both) |
| Link won't come up | Fiber polarity reversed | LC duplex — swap Tx/Rx fibers |
| High pre-FEC BER | Low OSNR (fiber loss, amp degradation, connector) | Check span loss, clean connectors, verify EDFA gain |
| High pre-FEC BER | Excessive CD (shouldn't happen with DSP) | Check if CD exceeds DSP range (>50,000 ps/nm = very long link) |
| Pre-FEC BER fluctuating | PMD from environmental changes | Check aerial fiber, bridges, temperature-dependent routes |
| Sudden BER increase | Filter tightening (ROADM cascade) | Check ROADM express path count, filter concatenation |
| Post-FEC errors | Pre-FEC BER exceeded FEC threshold | Reduce rate (modulation order) or fix physical layer |
| Module overheating | Insufficient airflow, adjacent high-power modules | Check line card cooling, ambient temp, module spacing |
| Module not recognized | Firmware/software version mismatch | Check platform compatibility matrix, update optics firmware |
| Wavelength collision | Two modules on same frequency | Audit wavelength plan, check both ends + any pass-through nodes |

### Methodology

**Coherent Link Troubleshooting — 8 Steps:**

```
1. Verify configuration (both ends)
   └─ Frequency, mode, power — do they match?

2. Check module state
   └─ Controller Up? Laser On? Module recognized?

3. Check received power
   └─ Within spec? If very low → fiber/connector issue

4. Check OSNR
   └─ If low → span loss, EDFA issue, in-band crosstalk

5. Check pre-FEC BER
   └─ High but stable → OSNR margin issue
   └─ High and fluctuating → PMD, vibration, filter narrowing

6. Check CD and DGD readings
   └─ CD way off expected? → Wrong fiber route, DCF in path (legacy)
   └─ DGD high? → PMD-limited fiber, aerial segment

7. Check FEC correction rate
   └─ Corrected codewords trending up → degrading, set threshold alarms

8. Check adjacent channels
   └─ Problem on one channel only → per-channel issue (filter, connector)
   └─ All channels degraded → common cause (fiber cut, amp failure, bend loss)
```

### The FEC Cliff

The most important concept in coherent troubleshooting:

```
Post-FEC BER
    │
    │  
    │  ← Zero errors (good)
    │  
    │  
    │──────────────────────────────────── ← FEC CLIFF
    │                                ╱
    │                              ╱
    │                            ╱   ← Errors flood in
    │                          ╱
    │                        ╱
    └──────────────────────────────────── Pre-FEC BER
                            ▲
                     FEC threshold
                    (e.g., 1.5×10⁻²)
```

**Below the cliff**: Pre-FEC BER can be terrible (10⁻³) and post-FEC is zero. Everything's fine.
**At the cliff**: A tiny degradation in pre-FEC BER causes post-FEC to go from zero to catastrophic.
**There is no gradual degradation** — it's a cliff, not a slope.

> **Operational rule**: Never run with less than 3 dB of OSNR margin above the FEC threshold. The cliff gives you no warning. One connector degradation, one EDFA aging, one hot day on aerial fiber — and you go from zero errors to link down.

### War Stories

> **The Midnight Frequency Drift**
>
> A major metro provider deployed 400ZR between two DCI sites — 80 km, clean fiber, ran perfectly for 3 months. Then, every night between 2-5 AM, pre-FEC BER would spike to 10⁻², barely below the FEC cliff. By 7 AM, it cleared. Always the same timeframe.
>
> Initial theory: Temperature cycling on aerial fiber causing PMD. Except the route was entirely underground conduit.
>
> Root cause: The building HVAC shut down at midnight. The router line card hosting the 400ZR module would slowly heat up as the data center's hot aisle temperature rose. The coherent module's tunable laser experienced **frequency drift** as its temperature exceeded the thermoelectric cooler's (TEC) ability to hold the setpoint. The drift was only ~1-2 GHz — well within the 75 GHz channel, but enough to shift partially out of the receiving ROADM's passband. The ROADM filter was slightly off-center (installed with 5 GHz offset — within spec but not ideal).
>
> Fix: Re-center the ROADM express port filter AND add line card inlet temperature monitoring with alarms at 45°C. The real lesson: coherent modules are precision instruments — they care about millidegrees of temperature stability. Treat them like the lasers they are, not like copper SFPs.

## Design Considerations

### Wavelength Planning for Coherent Networks

| Spacing | Capacity (C-band) | Use Case | Notes |
|---------|-------------------|----------|-------|
| 50 GHz | 96 channels × 200G | Legacy DWDM, 100G coherent | Standard 10G/40G/100G grid |
| 75 GHz | 64 channels × 400G | 400ZR | Fixed grid, OIF standard |
| 100 GHz | 48 channels × 400G | Conservative, easy planning | Wastes spectrum but simpler |
| 150 GHz | 32 channels × 800G | 800G single-carrier | Required for high-baud 800G |
| Flex-grid | Variable | Mixed rates, maximize capacity | Requires flex-grid ROADMs |

**C-band capacity at max fill:**
- 96 × 100G (50 GHz) = 9.6 Tb/s per fiber
- 64 × 400G (75 GHz) = 25.6 Tb/s per fiber
- 32 × 800G (150 GHz) = 25.6 Tb/s per fiber
- C+L band doubles all numbers

> **Key insight**: 800G per wavelength doesn't increase total fiber capacity vs 400G if it requires wider spacing. The win is **fewer wavelengths to manage** (32 vs 64), not more total bandwidth. C+L band operation is what actually doubles capacity.

### When to Use Which Technology

| Scenario | Technology | Why |
|----------|-----------|-----|
| DCI < 80 km | 400ZR QSFP-DD | Cheapest, standard, pluggable |
| DCI 80-120 km | 400ZR or ZR+ | ZR at limit; ZR+ adds margin |
| Metro ring 200-500 km | ZR+ (amplified) | Pluggable, flexible rate, over amp'd spans |
| Regional 500-1500 km | ZR+ CFP2-DCO or line system | CFP2 for router slot, line system for max reach |
| Long-haul > 1500 km | Line system (800G) | Proprietary FEC for max reach, regen if needed |
| Submarine | Line system (custom) | Extreme reach, specialized amplification |
| Campus/building | Direct detect (DR4/FR4) | Coherent is overkill under 2 km |

### Cost Model

The economic shift driving IPoDWDM adoption:

```
Traditional (gray optic + transponder):
  Router port:          $X
  Gray 400G optic:      $Y
  Transponder (400G):   $3Y–5Y
  OTN shelf slot:       $Z
  Management license:   $W
  ─────────────────────────
  Total per 400G:       $X + $Y + $3-5Y + $Z + $W

IPoDWDM (coherent pluggable):
  Router port:          $X
  400ZR optic:          $2-3Y
  ─────────────────────────
  Total per 400G:       $X + $2-3Y

Savings: 40-60% TCO reduction for DCI/metro
```

The transponder isn't free — but the 400ZR optic costs more than a gray optic. The savings come from eliminating the transponder shelf, its power, its cooling, its management, and the gray optic that connected to it.
