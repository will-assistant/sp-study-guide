## Lab Overview

This lab builds a closed-loop automation system: stream SR-TE policy and interface statistics via gNMI, detect congestion thresholds, and automatically update SR-TE candidate-path preferences to steer traffic away from overloaded links — all without a full PCE controller.

This is the "poor-man's controller" pattern — genuinely useful when you have SR-TE already deployed but no Crosswork/Paragon budget. Real-world carriers run variations of exactly this.

**What you will build:**
1. gNMI dial-out streaming from two IOS-XR routers (P1, P2)
2. gnmic collector receiving and parsing SR-TE policy stats + interface counters
3. Python automation daemon reading from gnmic stdout/Prometheus → evaluating thresholds
4. Automated NETCONF config push to flip SR-TE candidate-path preferences when a link exceeds 80% utilization
5. Validation showing traffic shifting across paths

## Part 2 — Streaming Telemetry Configuration

### 2.1 gnmic Configuration File

Create `/etc/gnmic/gnmic.yaml` on the automation host:

```yaml
# /etc/gnmic/gnmic.yaml
# gnmic collector: subscribes to SR-TE + interface paths on PE1/PE2

targets:
  pe1:
    address: 192.168.100.1:57400
    username: telemetry
    password: "changeme"
    insecure: true    # set to false + add tls stanza for production
    timeout: 10s
    subscriptions:
      - interface-counters
      - sr-te-policy-state
      - sr-te-policy-stats

  pe2:
    address: 192.168.100.2:57400
    username: telemetry
    password: "changeme"
    insecure: true
    timeout: 10s
    subscriptions:
      - interface-counters

subscriptions:
  # Interface counters — 30-second sample rate
  interface-counters:
    paths:
      - /interfaces/interface[name=GigabitEthernet0/0/0/0]/state/counters/in-octets
      - /interfaces/interface[name=GigabitEthernet0/0/0/0]/state/counters/out-octets
      - /interfaces/interface[name=GigabitEthernet0/0/0/0]/state/counters/in-pkts
      - /interfaces/interface[name=GigabitEthernet0/0/0/0]/state/counters/out-pkts
      - /interfaces/interface[name=GigabitEthernet0/0/0/1]/state/counters/in-octets
      - /interfaces/interface[name=GigabitEthernet0/0/0/1]/state/counters/out-octets
    mode: stream
    stream-mode: sample
    sample-interval: 30s
    encoding: proto

  # SR-TE policy state — ON_CHANGE fires on state transitions
  # CAVEAT: ON_CHANGE support for this oper path varies by IOS-XR release.
  # If you get a "MODE_NOT_ALLOWED" error, change stream-mode to "sample"
  # with a sample-interval of 30s. Use "gnmic capabilities" to check
  # supported modes for this path on your version.
  sr-te-policy-state:
    paths:
      - /Cisco-IOS-XR-infra-xtc-agent-oper:xtc/policies/policy
    mode: stream
    stream-mode: on-change     # Requires IOS-XR 7.3.2+ on some platforms; fall back to sample if rejected
    encoding: proto

  # SR-TE policy counters — 60-second sample
  sr-te-policy-stats:
    paths:
      - /Cisco-IOS-XR-infra-xtc-agent-oper:xtc/policies/policy[policy-name=TO_PE2]/policy-name
      - /Cisco-IOS-XR-infra-xtc-agent-oper:xtc/policies/policy[policy-name=TO_PE2]/forward-class
      - /Cisco-IOS-XR-infra-xtc-agent-oper:xtc/policies/policy[policy-name=TO_PE2]/active-candidate-path/preference
    mode: stream
    stream-mode: sample
    sample-interval: 60s
    encoding: proto

outputs:
  prometheus:
    type: prometheus
    listen: ":9273"
    expiration: 60s
    metric-prefix: gnmic
    append-subscription-name: true

  stdout-debug:
    type: file
    filename: /tmp/gnmic-debug.log
    format: event       # gnmic event format: timestamp, tags, values per update

# Key: subscription profiles are defined in subscriptions: and referenced
# by name under each target's subscriptions: list. This is the correct
# gnmic config pattern — no separate target-subscriptions block needed.
```

> **Path caveat:** IOS-XR SR-TE operational YANG paths vary by software release. The paths above (`Cisco-IOS-XR-infra-xtc-agent-oper:xtc/policies/policy`) are representative for IOS-XR 7.x. Always validate against your running version with:

> For OpenConfig-compatible paths (if OC SR-TE model is supported on your platform):

## Part 3 — Interface Utilization Calculation

Raw gNMI delivers cumulative octets, not utilization percentage. The automation daemon needs to delta-calculate bandwidth utilization between samples.

### 3.1 Utilization Math

```
util_percent = ((octets_delta * 8) / (sample_interval_seconds * interface_bps)) * 100

Example for Gi0/0/0/0 at 1 Gbps (1,000,000,000 bps), 30s sample interval:
  octets_delta = 350,000,000  →  util = (350M * 8) / (30 * 1G) = 9.33%
```

## Part 4 — Python Automation Daemon

### 4.1 Overview

The daemon:
1. Polls the local Prometheus endpoint exposed by gnmic
2. Calculates interface utilization from octet deltas
3. If PE1 Gi0/0/0/0 (P1 link) exceeds threshold → push NETCONF to PE1 to demote pref-200 path
4. If utilization drops back below lower threshold → restore preference

### 4.2 Full Daemon Code

Create `/opt/srte-daemon/srte_daemon.py`:

```python
#!/usr/bin/env python3
"""
SR-TE Automation Daemon
Reads gnmic Prometheus metrics → detects congestion → 
updates SR-TE candidate-path preference via NETCONF.

Requires: pip install requests ncclient pyyaml
"""

import time
import logging
import requests
import lxml.etree
from ncclient import manager
from typing import Optional

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s %(levelname)s %(message)s"
)
log = logging.getLogger("srte-daemon")

# --- Config ---
PROMETHEUS_URL = "http://localhost:9273/metrics"
POLL_INTERVAL_SEC = 30
INTERFACE_BPS = 1_000_000_000   # 1 Gbps — adjust to match lab interface
SAMPLE_INTERVAL_SEC = 30        # Must match gnmic sample-interval

NETCONF_HOST = "192.168.100.1"
NETCONF_PORT = 830
NETCONF_USER = "admin"
NETCONF_PASS = "changeme"

# Thresholds
CONGEST_THRESHOLD_PCT = 80.0    # Demote primary path above this
CLEAR_THRESHOLD_PCT = 50.0      # Restore primary path below this

# SR-TE state
PREF_PRIMARY = 200              # Normal preference for primary path
PREF_DEMOTED = 50               # Preference when demoted (below backup pref-100)
POLICY_NAME = "TO_PE2"
INTERFACE_LABEL = "GigabitEthernet0/0/0/0"  # Primary uplink to P1

# Internal state
_prev_out_octets: Optional[float] = None
_demoted = False                # True when primary path is currently demoted
# CAVEAT: _demoted is in-process state only. If the daemon crashes while the router
# is in demoted state (pref-50 active), restarting the daemon will reset _demoted=False
# and attempt to demote again — trying to delete pref-200 which no longer exists.
# Production fix: query router state at startup (NETCONF get-config) to sync _demoted.
# For this lab, monitor daemon logs and restart-proof with a simple state file if needed.


def fetch_metric(metric_name: str, labels: dict) -> Optional[float]:
    """Fetch a single Prometheus metric value from gnmic endpoint."""
    try:
        resp = requests.get(PROMETHEUS_URL, timeout=10)
        resp.raise_for_status()
    except requests.RequestException as e:
        log.warning(f"Prometheus fetch failed: {e}")
        return None

    for line in resp.text.splitlines():
        if line.startswith("#"):
            continue
        if not line.startswith(metric_name):
            continue
        # Check labels match
        match = True
        for k, v in labels.items():
            if f'{k}="{v}"' not in line:
                match = False
                break
        if match:
            try:
                return float(line.split()[-1])
            except (ValueError, IndexError):
                continue
    return None


def calc_utilization(current_octets: float) -> Optional[float]:
    """Calculate percent utilization from current cumulative octet counter."""
    global _prev_out_octets
    if _prev_out_octets is None:
        _prev_out_octets = current_octets
        log.info("Initializing octet baseline — utilization available next cycle.")
        return None

    delta = current_octets - _prev_out_octets
    _prev_out_octets = current_octets

    if delta < 0:
        # Counter wrapped or router reloaded
        log.warning("Octet counter wrapped — skipping this cycle.")
        return None

    bits_per_sec = (delta * 8) / SAMPLE_INTERVAL_SEC
    util_pct = (bits_per_sec / INTERFACE_BPS) * 100
    return util_pct


def build_srte_preference_config(policy_name: str, preference: int, new_preference: int) -> str:
    """
    Build NETCONF <edit-config> payload to change SR-TE candidate-path preference.
    Uses IOS-XR native YANG (Cisco-IOS-XR-infra-xtc-agent-cfg).
    
    NOTE: This modifies the preference of the explicit path entry named
    TO_PE2_VIA_P1 from `preference` to `new_preference`.
    Validate exact YANG path against your IOS-XR version.
    """
    # IMPORTANT: The IOS-XR SR-TE config YANG (Cisco-IOS-XR-infra-xtc-agent-cfg)
    # AUGMENTS the /sr container from Cisco-IOS-XR-segment-routing-ms-cfg.
    # The XML root element MUST be <sr> with the segment-routing-ms-cfg namespace.
    # Using <segment-routing xmlns="...infra-xtc-agent-cfg"> will produce an
    # "unknown element" RPC error — the most common gotcha for this lab.
    #
    # ALSO: IOS-XR NETCONF traditionally requires candidate datastore + commit.
    # This function returns the XML; push_netconf_config() handles candidate+commit.
    return f"""
<config xmlns:xc="urn:ietf:params:xml:ns:netconf:base:1.0">
  <sr xmlns="http://cisco.com/ns/yang/Cisco-IOS-XR-segment-routing-ms-cfg">
    <traffic-eng xmlns="http://cisco.com/ns/yang/Cisco-IOS-XR-infra-xtc-agent-cfg">
      <policies>
        <policy>
          <policy-name>{policy_name}</policy-name>
          <candidate-paths>
            <preferences>
              <preference xc:operation="delete">
                <path-index>{preference}</path-index>
              </preference>
              <preference xc:operation="create">
                <path-index>{new_preference}</path-index>
                <path-infos>
                  <path-info>
                    <type>explicit</type>
                    <hop-type>mpls</hop-type>
                    <segment-list-name>TO_PE2_VIA_P1</segment-list-name>
                  </path-info>
                </path-infos>
              </preference>
            </preferences>
          </candidate-paths>
        </policy>
      </policies>
    </traffic-eng>
  </sr>
</config>
"""


def push_netconf_config(xml_config: str) -> bool:
    """Push NETCONF edit-config to PE1 via candidate datastore. Returns True on success.

    IOS-XR NETCONF workflow:
      1. edit-config to candidate datastore
      2. commit — applies to running

    Using target="running" directly requires :writable-running capability,
    which is not available on all IOS-XR 7.x versions. Candidate+commit is
    the universally safe approach.
    """
    try:
        with manager.connect(
            host=NETCONF_HOST,
            port=NETCONF_PORT,
            username=NETCONF_USER,
            password=NETCONF_PASS,
            hostkey_verify=False,   # Lab only — enable in production with proper host key
            allow_agent=False,       # Disable SSH agent auth (avoids timeouts in lab)
            look_for_keys=False,     # Disable SSH key file search
            device_params={"name": "iosxr"},  # IOS-XR NETCONF handler for commit semantics
            timeout=30,
        ) as m:
            log.info(f"NETCONF session opened to {NETCONF_HOST}")
            # Parse XML string to element — use lxml directly (ncclient accepts element or string)
            config_element = lxml.etree.fromstring(xml_config.encode("utf-8"))
            edit_reply = m.edit_config(target="candidate", config=config_element)
            if not edit_reply.ok:
                log.error(f"NETCONF edit-config to candidate failed: {edit_reply.errors}")
                return False
            commit_reply = m.commit()
            if commit_reply.ok:
                log.info("NETCONF edit-config + commit OK")
                return True
            else:
                log.error(f"NETCONF commit failed: {commit_reply.errors}")
                # Attempt discard-changes to clean up candidate
                try:
                    m.discard_changes()
                except Exception:
                    pass
                return False
    except Exception as e:
        log.error(f"NETCONF error: {e}")
        return False


def demote_primary_path():
    """Drop primary path preference from PREF_PRIMARY to PREF_DEMOTED — traffic shifts to pref-100 backup."""
    global _demoted
    log.warning(
        f"[CONGEST] {INTERFACE_LABEL} over threshold — "
        f"demoting primary path preference {PREF_PRIMARY} → {PREF_DEMOTED}"
    )
    cfg = build_srte_preference_config(POLICY_NAME, preference=PREF_PRIMARY, new_preference=PREF_DEMOTED)
    if push_netconf_config(cfg):
        _demoted = True
        log.info("[ACTION] Primary path demoted — traffic should shift to backup path (pref 100)")
    else:
        log.error("[ACTION FAILED] NETCONF push failed — manual intervention required")


def restore_primary_path():
    """Restore primary path preference from PREF_DEMOTED back to PREF_PRIMARY."""
    global _demoted
    log.info(
        f"[CLEAR] {INTERFACE_LABEL} below clear threshold — "
        f"restoring primary path preference {PREF_DEMOTED} → {PREF_PRIMARY}"
    )
    cfg = build_srte_preference_config(POLICY_NAME, preference=PREF_DEMOTED, new_preference=PREF_PRIMARY)
    if push_netconf_config(cfg):
        _demoted = False
        log.info(f"[ACTION] Primary path restored to preference {PREF_PRIMARY}")
    else:
        log.error("[ACTION FAILED] NETCONF push failed — manual intervention required")


def main():
    log.info("SR-TE Automation Daemon starting...")
    log.info(f"  Target: {NETCONF_HOST}, Policy: {POLICY_NAME}")
    log.info(f"  Interface: {INTERFACE_LABEL} @ {INTERFACE_BPS/1e9:.1f}G")
    log.info(f"  Congest threshold: {CONGEST_THRESHOLD_PCT}%  Clear threshold: {CLEAR_THRESHOLD_PCT}%")

    while True:
        try:
            # Fetch current cumulative octets from Prometheus/gnmic
            # NOTE: gnmic metric name format is {prefix}_{subscription_name}_{leaf_name}.
            # With prefix="gnmic" and subscription="interface-counters", leaf="out-octets":
            #   → "gnmic_interface_counters_out_octets"
            # Label key/value format (interface name) depends on gnmic version — check:
            #   curl -s http://localhost:9273/metrics | grep gnmic | head -20
            # gnmic generates Prometheus labels from YANG list key names.
            # For OpenConfig /interfaces/interface[name=...], the YANG key is "name"
            # — so gnmic produces label key="name", NOT "interface_name".
            # Always verify actual label names with:
            #   curl -s http://localhost:9273/metrics | grep gnmic | head -5
            out_octets = fetch_metric(
                "gnmic_interface_counters_out_octets",
                labels={
                    "source": "pe1",
                    "name": INTERFACE_LABEL,   # YANG list key is "name", not "interface_name"
                }
            )

            if out_octets is None:
                log.warning("Could not fetch octet metric — check gnmic is running")
            else:
                util_pct = calc_utilization(out_octets)

                if util_pct is not None:
                    log.info(f"[METRIC] {INTERFACE_LABEL} utilization: {util_pct:.2f}%  (demoted={_demoted})")

                    if not _demoted and util_pct >= CONGEST_THRESHOLD_PCT:
                        demote_primary_path()
                    elif _demoted and util_pct < CLEAR_THRESHOLD_PCT:
                        restore_primary_path()
                    else:
                        log.debug(f"[METRIC] No action needed at {util_pct:.2f}%")

        except KeyboardInterrupt:
            log.info("Daemon stopped by user.")
            break
        except Exception as e:
            log.error(f"Unexpected error in main loop: {e}")

        time.sleep(POLL_INTERVAL_SEC)


if __name__ == "__main__":
    main()
```

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| `gnmic` exits immediately | Bad credentials or GRPC not enabled | Verify `show grpc trace`; confirm username/password |
| No Prometheus metrics at `:9273` | gnmic not started or subscription error | Check `gnmic` stdout for subscription errors |
| Metrics appear but utilization always 0% | Interface name label mismatch in `fetch_metric()` | Print raw Prometheus output; adjust label escaping |
| NETCONF fails with "unknown element" | XML namespace or root element wrong — most common for this lab | Verify root is `<sr xmlns="...segment-routing-ms-cfg">` not `<segment-routing xmlns="...infra-xtc-agent-cfg">` |
| NETCONF push fails with `<rpc-error>` | YANG path wrong for your IOS-XR version | Check `netconf-yang` CLI on router; compare YANG schema. Also confirm candidate datastore is enabled. |
| Metrics always 0 / daemon never triggers | Prometheus label key wrong | Run `curl localhost:9273/metrics \| grep gnmic \| head` and verify actual label key names match daemon code |
| Preference change applied but traffic doesn't shift | Forwarding plane not syncing | Check `show mpls forwarding`; check CEF adjacencies |
| Daemon demotes then immediately restores | Hysteresis insufficient — clear threshold too close to congest | Widen gap between `CONGEST_THRESHOLD_PCT` and `CLEAR_THRESHOLD_PCT` |
| Counter delta is negative | Router reload or counter wrap | Daemon handles this — skips one cycle, reseeds baseline |

## War Story: The Ghost Reroute

A real-world variant of this loop was deployed at a Tier-1 provider. Engineers set `CONGEST_THRESHOLD = 80%` with a 30-second poll interval and a 50% clear threshold — reasonable numbers. The problem: the primary link had a 24-hour traffic pattern where utilization oscillated between 74% and 82% every few minutes during prime time.

The daemon would demote the primary path at 82%, traffic shifted to backup. With reduced load on primary, utilization dropped below 50%... then restored, traffic shifted back, primary spiked to 82% again. The cycle repeated every ~4 minutes for 6 hours.

Impact: At every flip, PE1 re-imposes a different binding SID label stack (the headend changes from `{16003, 16002}` to `{16004, 16002}` and back). Transit routers (P1/P2) continue normal MPLS swap operations — the "rewrite" is at the headend PE only. However, each flip triggers a brief traffic disruption during forwarding plane reconvergence, plus BGP next-hop re-resolution overhead on high-RIB routers and packet reordering for long-lived flows as they switch paths mid-stream.

**Fix applied:** Added a minimum dwell timer — 5 minutes before either demoting or restoring — and widened clear threshold to 40%. Also added a cap: no more than 2 state flips per 30-minute window before alerting for human review.

**Lesson:** Closed-loop automation without hysteresis and rate-limiting is a congestion oscillator wearing an automation costume.

## Stretch Goals

### Stretch 1 — Dead Man's Switch
Add a watchdog: if gNMI data stops arriving for >5 minutes (gnmic crash, network loss), the daemon should automatically restore to policy defaults (preference 200 = primary) and send an alert, rather than leaving the network in a potentially sub-optimal demoted state indefinitely.

**Hint:** Track `last_metric_timestamp`; if `time.time() - last_metric_timestamp > 300`, call `restore_primary_path()` unconditionally.

### Stretch 2 — Multi-Link Monitoring
Extend the daemon to monitor both P1 and P2 uplinks simultaneously. Build a state machine:
- Both links healthy → pref-200 primary active
- P1 congested → shift to P2
- P2 congested → shift to P1 or ECMP
- Both congested → alert + no change (pick lesser evil)

**Hint:** Add a `LinkState` dataclass tracking utilization, dwell timer, and flip count per interface.

### Stretch 3 — Metric Export + Alerting
Replace print-based logging with Prometheus metrics from the daemon itself:
- `srte_automation_active_preference{policy="TO_PE2"}` — current active preference
- `srte_automation_flip_total{policy="TO_PE2", direction="demote|restore"}` — flip counter
- `srte_automation_util_pct{interface="Gi0/0/0/0"}` — computed utilization

Use `prometheus_client` library. Build a Grafana panel showing utilization vs flip events over time.

### Stretch 4 — Junos Variant
Port the NETCONF config push to work against Junos devices. Junos uses `netconf-yang` via Junos-native YANG (`junos-conf-root`) or PyEZ. Key difference: Junos edit-config uses `<configuration>` element, not IOS-XR's namespace. Implement and test both paths with a config flag to select vendor.

### Stretch 5 — Replace NETCONF with PCEP
Rather than pushing NETCONF config changes, integrate with an open-source PCE (e.g., pathman-sr or a local pce via ODL/OpenDaylight). Have the daemon trigger a PCUpdate message via the PCEP library instead of NETCONF. This mimics what Crosswork/Paragon does internally.
